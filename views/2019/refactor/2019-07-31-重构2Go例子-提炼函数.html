重构2Go例子：提炼函数

djhs0 2019-7-31 重构 go

# 提炼函数

提炼函数作为重构2书中第一个提到的重构手法，在编程实践中很频繁的使用这个手法来改善代码，而且提炼函数也是比较容易驾驭的一种重构手法。

# 何时需要提炼函数？

作者列出了几种看法：

  1. 函数应该在一屏之中能完全显示，如果超出则需要考虑提炼函数了
  2. 只要被使用不止一次的代码，就应该放进一个函数中

而作者给出了自己的观点：将意图与实现分开，如果需要花时间浏览一段代码才能弄清它到底在干什么，那么就应该将其提炼到一个函数中。

# 大量的函数调用影响性能？

现在编译器的优化功能已经将其影响降到很低了，同时谨记性能优化的一般指导方针：不要过早的优化性能

# 函数名

函数名需要有个好名字，一定要有自解释性，如果你找不到一个好的名字来描述这个函数，那么就要仔细考虑下是否有必要提炼函数。

# 实施注意点

提炼函数中，提炼部分被赋值的局部变量太多，这时最好先放弃提炼。考虑其他重构手法，拆分变量，以查询取代临时变量。

# 例子

# 原始函数

// Invoice 发票
type Invoice struct {
	Customer string
	Orders   []int32
}

// PrintOwing 打印未付账单
func PrintOwing(invoice Invoice) {
	fmt.Println("***************************")
	fmt.Println("***** Customer Owes *******")
	fmt.Println("***************************")

	var outstanding int32

	for _, val := range invoice.Orders {
		outstanding += val
	}

	// print details
	fmt.Printf("name: %s\n", invoice.Customer)
	fmt.Printf("amount: %d\n", outstanding)
}

# 提炼后的函数

// PrintOwing2 提炼函数后
func PrintOwing2(invoice Invoice) {
	PrintBanner()

	outstanding := CalcOutstanding(invoice)

	PrintDetail(invoice, outstanding)
}

// PrintBanner 打印横幅
// 入参：无  出参：无
func PrintBanner() {
	fmt.Println("***************************")
	fmt.Println("***** Customer Owes *******")
	fmt.Println("***************************")
}

// CalcOutstanding 计算出未付款项
// 入参：有  出参：有
func CalcOutstanding(invoice Invoice) int32 {
	var result int32
	for _, val := range invoice.Orders {
		result += val
	}

	return result
}

// PrintDetail 打印账单的详细
// 入参：有  出参：无
func PrintDetail(invoice Invoice, outstanding int32) {
	fmt.Printf("name: %s\n", invoice.Customer)
	fmt.Printf("amount: %d\n", outstanding)
}

# 运行例子

func main() {
	order := []int32{1, 2, 3, 4}
	invo := Invoice{Customer: "djh", Orders: order}

	PrintOwing(invo)
	PrintOwing2(invo)
}
