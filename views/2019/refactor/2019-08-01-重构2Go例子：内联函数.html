    重构2Go例子：内联函数

    djhs0 2019-8-1 重构 go

    # 内联函数

    内联函数重构手法时作为提炼函数的反向重构手法。 在提炼函数重构手法中提到了提炼函数时需要注意的点，那么在实施的过程中不可避免的会出现提炼函数不合理的情况，这样就需要内联函数来解决其中一部分的问题。为什么说时一部分呢？因为内联函数解决的是如何将不合理的提炼函数重新内联到原始函数中的问题。

    看一个简单的例子，书本中的例子是一个使用了三元表达式的例子，由于go不支持三元表达式，所以需要使用if/else代替，而我不认为直接使用if/else比将其提炼到一个函数中更好，所以我根据自己理解改变/设计了一个例子。

    import "fmt"

// Driver 司机数据结构
type Driver struct {
	Name   string
	Number int32
}

// DriverInfo 司机的信息
func DriverInfo(driver Driver) string {
	return fmt.Sprintf("Name: %s Number: %d", GetDriverName(driver), driver.Number)
}

// GetDriverName 获取司机的名字
func GetDriverName(dirver Driver) string {
	return dirver.Name
}

// 重构后

// DriverInfo2 重构：司机的信息
func DriverInfo2(driver Driver) string {
	return fmt.Sprintf("Name: %s Number: %d", driver.Name, driver.Number)
}
