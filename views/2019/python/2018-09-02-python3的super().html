            # python3中的super()

            在python中的继承机制中，super使用的很多。使用super可以使写的程序更具扩展性。所以有必要对super()做一次细致的理解。

            # super的官方文档

            super([type[,object or type]]) 返回一个代理对象，该代理对象将方法调用委托给type的父类或者兄弟类。这对访问类中被重写的继承方法十分有用。搜索的顺序和getattr()一样，但是type本身被跳过。

            类型的__mro__属性列出了getattr()和super()都要使用的方法解析搜索顺序。这个属性是动态的，可以在继承层次结构更新时更改。

            如果省略第二个参数，则返回的超级对象是未绑定的。如果第二个参数是对象，则isinstance(obj, type)一定要为真。如果第二个参数是type，则issubclass(type2, type)一定要为真（这对classmethods十分有用）。

            # 使用方法

            class super(object)
	"""
	super() -> same as super(__class__, <first argument>)
 	super(type) -> unbound super object
 	super(type, obj) -> bound super object; requires isinstance(obj, type)
 	super(type, type2) -> bound super object; requires issubclass(type2, type)
 	Typical use to call a cooperative superclass method:
 	class C(B):
 	    def meth(self, arg):
 	        super().meth(arg)
 	This works for class methods too:
 	class C(B):
 	    @classmethod
 	    def cmeth(cls, arg):
 	        super().cmeth(arg)
	"""

            # 深入理解

            python是支持多继承的，如果是单继承，那么很好理解C继承B，super(C, self).method()实际上是调用B中的method方法。但是如果是多继承呢？C(A, B, ....)，那么super(C, self).method()到底调用是谁的方法呢？

            python为了解决这个多继承的问题，使用了MRO机制，也就是方法解析顺序。在super的官方解释中，可以发现出现了__mro__属性。

            类型的__mro__属性列出了getattr()和super()都要使用的方法解析搜索顺序。这个属性是动态的，可以在继承层次结构更新时更改。

            搜索的顺序和getattr()一样，但是type本身被跳过。

            getattr()是用作访问类中属性的方法。但是super有点不一样，type本身被跳过去了。解释下type本身被跳过去是什么意思。

            结论:

            在使用super(__class__, arg)时，python已以下步骤工作

            1. 求出arg的mro顺序，即arg.__class__.mro()。
            2. 在mro顺序中找到__class__，然后从这个__class__的下一个元素开始搜索方法属性。

            # 扩展，mro的计算

            MRO的计算方法是我在理解super的过程中找的一篇文章中提到了，名字是C3算法，文章地址我忘记保存了，如果有人知道原文章地址，可以在github中或邮件提醒我。

            # 新式类和老式类

            python3中完全使用新式类(MRO使用的C3算法，老式类使用的是BFS和DFS)，老式类用于3之前。

            # C3算法

            # 拓扑排序算法

            在图论中，对于有向无环图，所有顶点的线性序列满足以下情况：

            1. 每个顶点出现且只出现一次。
            2. 若存在一条顶点A到顶点B的路径，则在序列中A一定在B的前面。

            满足以上情况的序列，称之为该有向无环图的拓扑序列。
            只有有向无环图(DAG)才有拓扑序列，非DAG没有这一说法。

            # 算法步骤

            1. 在DAG图中，找一个没有前驱的顶点(入度为0)，输出之。
            2. 在图中删除这个顶点和与其相关的边。
            3. 重复1，2步，直到DAG图为空，或者不存在无前驱的顶点(表示此图不是DAg)

            # c3算法描述

            1. c3算法用来生成一个线性序列。
            2. 如果继承一个基类。
              class B(A): pass
              则B的mro为[B, A]
            3. 如果继承多个基类： class B(A1, A2, A3 ...): pass 则B的MRO为： mro(B) = [B] + merge(mro(A1), mro(A2), mro(A3), ..., [A1, A2, A3, ...])

            # merge操作

            meirge操作的算法类似于拓扑排序，但是拓扑排序得出的序列并不是唯一的，mro需要唯一的线性序列，所以，merge在其中加入了一些限制，使其能生成唯一的线性序列。

            # 简述

            1. 遍历执行merge操作的序列(保证了从第一个基类开始)
            2. 如果一个序列中的第一个元素在其他序列中也是第一个元素，或者不再其他序列中存在(类似于拓扑排序中的入度为0的顶点)，则将其合并到当前mro中。
            3. merge之后的序列，继续merge操作，直到序列为空。若序列不为空，则表明不合法。(类似于拓扑排序中的非DAG)

            # 例子

            """
	C3算法中的merge操作
"""

            # 公式步骤

            mro(G) = [G] + merge(mro(E), mro(F), [E, F])

            1. mro(E) = [E] + merge(mro(A), mro(B), [A, B])
            • mro(A) = [A, O]
            • mro(B) = [B, O]
            • mro(E) = [E] + merge([A, O], [B, O], [A, B])
            • mro(E) = [E, A] + merge([O], [B, O], [B])
            • mro(E) = [E, A, B] + merge([O], [O], []) # 虽然O在[O]中是第一个元素，但是在[B, O]不是第一个元素，所以不符合。
            • mro(E) = [E, A, B, O]
            1. mro(F) = [F] + merge(mro(B), mro(C), [B, C])
            • mro(B) = [B, O]
            • mro(C) = [C, O]
            • mro(F) = [F] + merge([B, O], [C, O], [B, C])
            • mro(F) = [F, B] + merge([O], [C, O], [C])
            • mro(F) = [F, B, C] + merge([O], [O], [])
            • mro(F) = [F, B, C, O]

            mro(G) = [G] + merge([E, A, B, O], [F, B, C, O], [E, F])
            mro(G) = [G, E] + merge([A, B, O], [F, B, C, O], [F])
            mro(G) = [G, E, A] + merge([B, O], [F, B, C, O], [F])
            mro(G) = [G, E, A, F] + merge([B, O], [B, C, O], [])
            mro(G) = [G, E, A, F, B] + merge([O], [C, O], [])
            mro(G) = [G, E, A, F, B, C] + merge([O], [O], [])
            mro(G) = [G, E, A, F, B, C, O]
            最终class G的方法解析顺序为[G, E, A, F, B, C, O]
