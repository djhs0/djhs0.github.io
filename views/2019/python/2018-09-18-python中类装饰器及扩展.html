      djhs0 2018-9-18 decorator

      # Python中使用类作为装饰器及扩展

      装饰器很容易对已存在的函数做功能扩展，这样不会破坏函数中原有的代码。使用起来很方便，但是有时候会对其理解不深导致出现错误。这篇文章源于我想写一个既能装饰普通函数又能装饰类中方法的装饰器。

      # 问题

      经过思考，如果使用函数装饰器的话，可以实现

      1. 装饰器带参数，形如@dec(args)这种。
      2. 闭包特性，能修改闭包域中变量。比如定义一个count计算函数调用次数。

      那么使用类装饰器的话，我想实现

      1. 装饰器带参数，形如@Dec(args)这种。
      2. 有一个变量可以记录函数调用次数。
      3. 能同时装饰普通函数和类中方法。

      # 不带参数的类装饰器

      在查找资料时，从python3-cookbook中提供了一种写法，可以同时装饰普通函数和类中方法。

      from types import MethodType
from functools import wraps

class Profiled:
    def __init__(self, func):
        wraps(func)(self)
        self.ncalls = 0

    def __call__(self, *args, **kwargs):
        self.ncalls += 1
        return self.__wrapped__(*args, **kwargs)

    def __get__(self, instance, cls):
        if instance is None:
            return self
        else:
            return MethodType(self, instance)

      # 带参数的类装饰器

      根据对装饰器原理的理解，我发现有一种可以同时实现上面三种需求的写法。

      # 代码

      from types import MethodType
from functools import wraps

class Dec():
    def __init__(self, text):
        self.text = text
    
    def __call__(self, func):
        class Decorator:
            def __init__(self, func, text):
                wraps(func)(self)
                self.text = text
                self.count = 0  # 记录函数运行几次
            def __call__(self, *args, **kwargs):
                self.count += 1
                print("--START", self.text)
                print("-COUNT:", self.count)
                res = self.__wrapped__(*args, **kwargs)
                print("----END", self.text)
                return res
            def __get__(self, instance, owner):
                if instance is None:
                    return self
                else:
                    return MethodType(self, instance)
        return Decorator(func, self.text)

      @Dec("普通函数")
def normal(name):
    print(name)

class Test():
    @Dec("方法")
    def function(self, name):
        print(name)
t = Test()
t.function("djh")

      # 原理简析

      如运行测试中的例子，可以将步骤分解下

      1. @Dec("普通函数")：将会执行创建Dec实例的动作，然后生成实例d。
      2. d(func)执行，将会调用Dec类中的__call__方法，然后返回值是Decorator的实例dd。
      3. 然后dd(name)调用，后面就是正常的cook-book中类装饰器用法了。

      # 免责

      实际上python中装饰器的实现依赖于闭包的特性，无论闭包函数能访问外部函数的变量，还是修改变量都是闭包的特性来实现的，但是上例中我的实现，其中的text是作为参数传进来的，而且count并不是属于外部函数，那么这个到底算不算装饰器呢？我也不确定，而且这个函数我没有经过完备的测试，只是符合自己的需求，所以对使用这个函数出现的事故我要扔锅了。不过如果你在使用中发现这个函数有bug，也希望你告诉我改一改。谢谢。
