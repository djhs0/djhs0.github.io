                # str,unicode,byte

                byte:字节，一个字节包含8个bit，1byte=8bit,一个字符是两个字节。

                ASCII码：使用8bit来表示英文字符和符号。
                字符和字节：
                ASCII码:一个英文字母占一个字节空间，中文汉字占两个字节。
                UTF-8:一个英文字母占一个字节空间，一个中文汉字占二到四个字节。
                unicode:一个英文占两个字节，一个中文占两个字节。
                符号：英文标点占一个字节，中文标点占两个字节。

                unicode:是字符集，utf-8是编码规则。
                字符集：为每一个字符分配一个唯一的ID，即码位，code point.
                编码规则：将码位转换成字节序列的规则。

                在使用utf-8(Unicode Transformation Format)转换unicode时实际上是将一个字符对应的unicode的ID找到一个utf-8对应的ID。
                那么这样做的目地是：

                1. 规范存储方式，如大小端。
                2. 变长编码，一定程度上压缩大小。

                # python中encode和decode

                1. encode:将unicode字符串按照一定的编码规则转换成字节序列。
                2. decode:将一串字节序列按照一定的方式解码成unicode字符串。
                >>> u_seq = "哈哈"
>>> b_seq = u_seq.encode()
>>> b_seq
b'\xe5\x93\x88\xe5\x93\x88'
>>> b_seq.decode()
'哈哈'
# 可以看到python3中字符串默认为unicode字符串，可以encode。

>>> b = "严"
>>> b.encode('utf-8')
b'\xe4\xb8\xa5'
>>> b.encode('utf-16')
b'\xff\xfe%N'
>>> b.encode('utf-32')
b'\xff\xfe\x00\x00%N\x00\x00'
>>> b.encode('ASCII')

                # utf-8编码规则

                1. 对于对于单字节字符，这个字节的第1位为0，后7位为unicode码，所以对于英文来讲utf-8编码和ASCII编码时一样的。
                2. 对于n字节字符，第一个字节的前n位为1，n+1位为0,后面n-1个字节的前两位都为10,其他位放置unicode码。
                Unicode符号范围(十六进制) UTF-8编码方式（二进制）
                0000 0000-0000 007F 0xxxxxxx
                0000 0080-0000 07FF 110xxxxx 10xxxxxx
                0000 0800-0000 FFFF 1110xxxx 10xxxxxx 10xxxxxx
                0001 0000-0010 FFFF 11110xxx 10xxxxxx 10xxxxxx 10xxxxxx

                # 大端小端问题

                大小端问题的出现是cpu寄存器大小超过1字节，即8位导致的，(对！不但系统有32/64位之分，cpu也有位数之分，32位cpu的寄存器是32位的，64位cpu同理)，当cpu的寄存器位数超过8位，比如16，32，那么存储存储字符的编码的时候就出现了大端和小端问题。
                大端：低位字节放内存的高地址位，高位字节放内存的低地址位。
                小端：高位字节放内存的高地址位，低位字节放内存的低地址位。

                下面是32bit宽数0x12345678在cpu中的存放方式。

                内存地址 小端模式 大端模式
                0x4000 0x78 0x12
                0x4001 0x56 0x34
                0x4002 0x34 0x56
                0x4003 0x12 0x78
                1. 从上面来看，如果cpu是从低位开始读取的时候，那么因为数字的符号放在这个数字的高位，所以cpu一开始就能判断这个数字的符号。

                # 资源

                unicode之痛：
                http://pycoders-weekly-chinese.readthedocs.io/en/latest/issue5/unipain.html
                从其中得到5个事实：

                1. 程序的所有的输入与输出都是byte。
                2. byte流中不会包含编码信息，即不会指出其是什么编码格式(utf-8/utf-16/ascii etc..)。
                3. 指明的编码可能是错误的。
                4. 程序必须处理byte和unicode。
                5. 世界上的文本中的符号数远远大于256，所以1字节8位不能表示完。

                # Point

                1. 在计算机内存中同一使用unicode编码，在需要保存到内存或者传输的时候，就转成各种编码方式生成的编码。
                2. python的ord：获取字符的整数，chr:获取整数的字符
                >>> '\u4e00'
'一'
>>> '\u9fa5'
'龥'
>>> ord('-')
45
>>> ord('一')
19968
>>> ord('严')
20005
>>> chr(20005)
'严'
