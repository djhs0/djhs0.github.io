        # type和object

        python中使用class关键字定义类，使用__new__函数来创建类的实例，那么类是由谁创建的呢？使用class只是定义了类，实际上创建类是由python中type来完成的。

        class A:
    arg = 1
等价于
A = type("A", (), {"arg": 1})

        # type和object疑问

        • 问题：
        1. 为什么object的类型是type？也就是说object是元类type创建的。
        2. 为什么type的父类是object？
        3. type不存在的时候object也不存在，而生成type的时候需要继承object，但是此时object并不存在。
        • 答：
        1. 以上问题可以类比成是鸡生蛋，还是蛋生鸡的问题。
        2. 理解就是蛋和鸡一开始就是混沌状态，鸡不是鸡，蛋也不是蛋，最终鸡是鸡的时候，蛋也是蛋了。
        3. finally：python的type和object都是在python解释器启动的时候就为type和object分配好了内存，只是在初始化的时候将它们各自引用了起来（type的父类是object，object是由type创建的），这样就好像是文件删除中的逻辑删除,即没有真正上删除，只是我声明已经将文件删除了。python的type和object也是这样，我并没有真正的使用type来创建object，也没有真正的用type来继承object，只是初始化时，我声明来它们相互引用。
